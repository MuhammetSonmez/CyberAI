[CLS] symmetric encryption [SEP] Symmetric encryption uses the same key for both encryption and decryption. [SEP]
[CLS] asymmetric encryption [SEP] Asymmetric encryption uses a public key for encryption and a private key for decryption. [SEP]
[CLS] hash functions [SEP] Hash functions convert data into a fixed-size hash value, often used for data integrity checks. [SEP]
[CLS] digital signatures [SEP] Digital signatures use asymmetric cryptography to verify the authenticity of a digital message or document. [SEP]
[CLS] cryptographic protocols [SEP] Cryptographic protocols like SSL/TLS provide secure communications over a computer network. [SEP]
[CLS] key exchange algorithms [SEP] Key exchange algorithms like Diffie-Hellman allow two parties to establish a shared secret over an insecure channel. [SEP]
[CLS] public key infrastructure [SEP] Public key infrastructure (PKI) manages keys and certificates for secure electronic transfer of information. [SEP]
[CLS] encryption algorithms [SEP] Encryption algorithms like AES and DES are used to encrypt data. [SEP]
[CLS] block ciphers [SEP] Block ciphers encrypt data in fixed-size blocks, typically 64 or 128 bits. [SEP]
[CLS] stream ciphers [SEP] Stream ciphers encrypt data one bit or byte at a time, often used in real-time data encryption. [SEP]
[CLS] cryptographic hash algorithms [SEP] Cryptographic hash algorithms like SHA-256 are used to create a unique hash for data. [SEP]
[CLS] message authentication codes [SEP] Message authentication codes (MACs) provide data integrity and authenticity. [SEP]
[CLS] random number generation [SEP] Random number generation is crucial for creating cryptographic keys. [SEP]
[CLS] elliptic curve cryptography [SEP] Elliptic curve cryptography (ECC) is based on the algebraic structure of elliptic curves over finite fields. [SEP]
[CLS] quantum cryptography [SEP] Quantum cryptography uses quantum mechanics principles to secure data against eavesdropping. [SEP]
[CLS] homomorphic encryption [SEP] Homomorphic encryption allows computation on ciphertexts, generating an encrypted result which, when decrypted, matches the result of operations performed on the plaintext. [SEP]
[CLS] zero-knowledge proofs [SEP] Zero-knowledge proofs allow one party to prove to another that a statement is true without revealing any information beyond the validity of the statement itself. [SEP]
[CLS] secure multi-party computation [SEP] Secure multi-party computation enables parties to jointly compute a function over their inputs while keeping those inputs private. [SEP]
[CLS] cryptographic salts [SEP] Cryptographic salts are random data that are used as additional input to a hash function to prevent dictionary attacks. [SEP]
[CLS] side-channel attacks [SEP] Side-channel attacks are techniques that can be used to extract information from cryptographic systems by analyzing physical implementations. [SEP]
